# ü§ù Contribution Guidelines for My Ford Benefits Portal

We appreciate your interest in contributing to the **My Ford Benefits Portal** project! Your contributions are vital to enhancing this resource.

## Contribution Guidelines

We maintain a high standard of code quality and documentation. All contributions must:
* Be well-tested.
* Adhere to the project's existing linting and formatting rules.
* Be clearly documented, especially for new components or complex logic.

## How to Contribute

To start contributing, please follow these steps:

1.  **Fork** the repository and clone it to your local machine.
2.  Create a dedicated branch for your feature or fix: `git checkout -b feature/new-benefit-card`.
3.  Implement your changes, ensuring no existing tests break and adding new tests as needed.
4.  Commit your changes using descriptive, conventional messages (see "Review Process").
5.  Push your branch to your forked repository.
6.  Open a **Pull Request (PR)** against the `main` branch of the original repository.

## Reporting Issues

If you find a bug, error, or data inconsistency:

* Open an issue on the **GitHub Issues** page.
* Use a clear title, such as `[BUG] X-Plan PIN Generator Fails on Safari`.
* Provide clear **steps to reproduce** the issue, the expected outcome, and the actual outcome.
* Include details about your operating system and browser version.

## Suggesting Features

We welcome ideas for new features, data visualizations, or improvements:

1.  Check existing issues to avoid duplication.
2.  Open a new issue with the label `enhancement`.
3.  Describe the feature and explain the value it brings to the user (e.g., "Allow filtering of Health Plans by deductible amount").

## Code of Conduct

All contributors are expected to abide by the project's Code of Conduct. We strive to maintain a respectful, harassment-free, and inclusive environment.

## Review Process

All contributions are reviewed by the core maintainers.

* **Commit Messages:** Should follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) format (e.g., `feat: implement dental plan card` or `fix: correct retirement match percentage`).
* **PR Description:** Should concisely summarize the changes and link to the relevant issue using keywords like `Closes #123`.
* **Feedback:** Reviewers will provide constructive feedback. Please address comments promptly.
